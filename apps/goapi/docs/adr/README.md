# Architecture Decision Records

ADRs capture significant, stable decisions: context, the choice made, and consequences.

## Naming

Use `NNN-short-title.md` (three-digit prefix), e.g. [001-http-package-layout.md](001-http-package-layout.md).

## Status

Typical values: **Proposed**, **Accepted**, **Superseded by ADR-XXX**, **Deprecated**.

## Index

| ADR | Title | Status |
|-----|--------|--------|
| [001](001-http-package-layout.md) | HTTP package layout under `internal/transport/http` | Accepted |

## New ADR

Copy [template.md](template.md), choose the next number, set status to Proposed, then update this index when accepted.
